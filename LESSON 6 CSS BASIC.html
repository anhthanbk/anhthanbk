1. Ba vị trí đặt CSS


Độ ưu tiên các vị trí đặt CSS: CSS inline ưu tiên đầu tiên, đối với CSS Internal và CSS External, cái nào khai báo sau sẽ ghi đè lên cái trước đó

a. Inline

CSS nội tuyến: Viết ngay bên trong phần tử của HTML

  • Khó thay đổi, mổi lần thay đổi phải thay đổi riêng lẻ
  • Không tái sử dụng được
  • Khó quản lý
  • Khó tách biệt cấu trúc

b. Internal

CSS nội bộ: là cách viết CSS ngày trong cặp Style (đặt trong khối HEAD)

  • Ưu điểm:
    • Code tách biệt với phần tử
    • có thể áp dụng với toàn trang
  • Nhược điểm
    • Vẫn khó quản lý khi dự án lớn
    • Quá nhiều mã CSS làm HTML trở nên cồng kềnh

c. External

CSS bên ngoài: Trong trường hợp này sẽ áp dụng cho toàn bộ file HTML nằm trên trang web, tránh việc phải copy CSS ra tất cả các file

Tạo file CSS -> khai báo trong tệp HTML để link đến tệp CSS -> rel="Tên file CSS" thông báo đến tệp HTML về tệp CSS áp dụng
